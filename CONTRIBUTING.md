# योगदान (Contributing) करने के लिए आपका स्वागत है!

हमारे [अपने प्रोजेक्ट का नाम] प्रोजेक्ट में योगदान देने के लिए धन्यवाद। हम आपकी मदद और समय की सराहना करते हैं!

नीचे दिए गए दिशानिर्देश (Guidelines) बताते हैं कि आप इस प्रोजेक्ट में कैसे योगदान दे सकते हैं।

## 1. बग रिपोर्ट करना (Reporting Bugs)

अगर आपको कोड में कोई गलती (Bug) मिलती है, तो कृपया Pull Request बनाने के बजाय पहले एक **Issue** खोलें।

* **स्पष्ट शीर्षक (Clear Title):** बग का स्पष्ट और छोटा विवरण दें।
* **दहराने के चरण (Steps to Reproduce):** बताएं कि बग कैसे उत्पन्न होता है। (जैसे: '1. यह बटन दबाएं', '2. यह इनपुट दें', '3. यह एरर आती है')
* **अपेक्षित व्यवहार (Expected Behavior):** बताएं कि कोड को वास्तव में क्या करना चाहिए था।
* **वास्तविक व्यवहार (Actual Behavior):** बताएं कि कोड क्या कर रहा है।
* **विवरण:** बताएं कि आप कौन सा Python वर्जन और ऑपरेटिंग सिस्टम उपयोग कर रहे थे।

## 2. नया फीचर सुझाना (Suggesting New Features)

नया फीचर सुझाने या किसी बड़े बदलाव पर चर्चा करने के लिए भी एक **Issue** खोलें।

* **समस्या:** आप कौन सी समस्या हल करना चाहते हैं?
* **समाधान:** आप क्या फीचर सुझा रहे हैं, और यह प्रोजेक्ट के लिए क्यों महत्वपूर्ण है?

## 3. कोड योगदान (Code Contributions via Pull Request)

यदि आप कोड में बदलाव करना चाहते हैं (जैसे बग फिक्स या नया फीचर), तो कृपया इन चरणों का पालन करें:

1.  **Fork करें:** इस Repository को अपने GitHub अकाउंट में Fork करें।
2.  **Clone करें:** अपने लोकल मशीन पर Repository को Clone करें।
3.  **नई ब्रांच बनाएँ:** अपने बदलावों के लिए एक नई ब्रांच बनाएँ। (उदाहरण के लिए: `feature/नया-फीचर` या `bugfix/बग-का-नाम`)
4.  **बदलाव करें:** अपने कोड में आवश्यक बदलाव करें।
5.  **जाँच (Test) करें:** बदलाव करने के बाद, सुनिश्चित करें कि आपका कोड ठीक से काम कर रहा है और उसने किसी मौजूदा फंक्शनलिटी को तोड़ा नहीं है।
6.  **Pull Request (PR) बनाएँ:** अपनी ब्रांच को यहाँ (मुख्य Repository) Pull Request के रूप में भेजें।

### Pull Request बनाते समय:

* **शीर्षक:** PR का शीर्षक आपके बदलावों को संक्षेप में समझाना चाहिए।
* **विवरण:** PR के विवरण में विस्तार से बताएं कि आपने क्या बदला है और क्यों।
* **Issue को लिंक करें:** यदि आपका PR किसी Issue को हल करता है, तो विवरण में `Closes #Issue-नंबर` जरूर लिखें (उदाहरण: `Closes #42`)।

## 4. कोड शैली (Code Style)

* कृपया **PEP 8** (Python Enhancement Proposal 8) के कोडिंग स्टैंडर्ड्स का पालन करें।
* कोड को स्पष्ट और आसान बनाने के लिए पर्याप्त **टिप्पणी (Comments)** का उपयोग करें।
